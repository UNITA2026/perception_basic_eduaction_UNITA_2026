* 정규민
  > [1월 2일] 초음파센서 및 라이다  
  이번 활동을 통해 라이다 데이터를 이용해 주변 장애물을 클러스터링하고, 초음파 센서 6개를 활용해 각 방향의 근접 장애물을 감지하여 음성으로 알리는 시스템을 구현해보았다.  
  라이다 기반 클러스터링으로 환경을 시각적으로 인식하고, 초음파 센서를 통해 근거리 안전 영역을 보완함으로써 센서 융합의 중요성을 체감할 수 있었다.  
  특히 각 센서 위치에 따라 서로 다른 음성을 출력하도록 구성하면서, 로봇이 주변 상황을 직관적으로 전달할 수 있다는 점이 인상 깊었다.  
  이번 활동을 통해 단일 센서에 의존하는 것보다 복수 센서를 결합했을 때 인식 신뢰도가 크게 향상된다는 것을 확인했다.  
  보완할 점으로는, 실환경에서 발생하는 노이즈와 오검출을 줄이기 위한 필터링 및 임계값 자동 조정 기법을 적용하고, 라이다 클러스터 결과와 초음파 데이터를 정량적으로 연계하는 센서 융합 알고리즘을 추가하여 시스템의 안정성과 환경 적응성을 더욱 높일 필요가 있다고 느꼈다.  
  [1월 6일] 카메라  
  본 활동에서는 카메라 영상을 RViz에 띄우고, 학습된 모델(.pt)을 이용해 신호등 색상을 인식한 뒤, 인식 결과를 mp3 음성과 시각화로 연동하는 시스템을 구현하였다.  
  이를 통해 인지 결과를 사용자에게 전달하는 전체 로봇 시스템 구조를 직접 경험할 수 있었다.  
  특히 카메라 설정, 토픽 연결, 노드 통합 과정에서 시스템 구성의 중요성을 체감하였다. 향후에는 다양한 환경에서도 안정적으로 동작하도록 실환경 대응 성능을 개선하고, 시간적 필터링과 신뢰도 기반 판단을 통해 오검출을 줄여 시스템 안정성을 향상시키고자 한다.
* 이원종
  > [1월 2일 초음파, 라이다 프로젝트]  
  이번 프로젝트를 통해 초음파 센서 이해하고, 아두이노와 ROS 간의 Serial 통신 설정하는 것을 경험했다. 특히 센서 데이터를 어떤 ROS 토픽 메시지 타입으로 정의하느냐에 따라, 이후 노드 간 연동과 활용 방식이 크게 달라진다는 점을 경험했다. 이를 통해 단순 값 전달이 아니라, 전체 시스템 흐름을 고려한 메시지 구조 설계의 중요성을 이해하게 되었다.  
  [1월 6일 카메라 신호등 인식 프로젝트]  
  이번 프로젝트를 통해 인식 모델의 성능뿐 아니라, 인식 결과를 실제 시스템에서 활용 가능한 형태의 토픽으로 구조화하는 과정이 중요하다는 점을 느꼈다. 특히 YOLO 기반 신호등 인식 결과를 Bool·String 토픽과 시각화 이미지로 분리해 퍼블리시하면서, 모듈 간 인터페이스 설계가 시스템 안정성과 직결된다는 것을 경험했다. 이를 통해 단순한 모델 구현을 넘어, 로봇 시스템 전체 관점에서 인식–전달–활용 흐름을 설계하는 역량을 기를 수 있었다.
* 김민서
  >  <1월 2일>  
  원형 구조물에 6개의 초음파 센서와 상단에 라이다 C1을 배치하여 물체 인식 시스템을 구현하였다. 초음파 센서가 물체를 감지하면 센서 번호에 따라 음성(TTS)으로 알림을 출력하고, 라이다를 이용해 물체를 클러스터링하여 RViz에서 시각화하는 것이 목표였다. 내가 맡은 역할은 RViz에서 하나의 물체가 여러 마커로 인식되는 문제를 해결하기 위해 일정 범위 내의 마커를 하나의 물체로 판단하는 로직을 팀원과 함께 구현하고, 6개의 초음파 센서 데이터를 하나로 묶어 publish한 뒤 각 센서에 맞는 TTS를 출력하는 기능을 만드는 것이었다. 여러 노드가 동시에 subscribe하는 구조가 처음이라 TTS 실행과 음성 품질 문제로 시행착오가 있었지만, 이를 통해 ROS 기반 협업 구조에 대한 이해를 높일 수 있었고, 센서 배치와 하드웨어 제작의 중요성도 함께 느낄 수 있었다.  
  <1월 6일>  
  처음으로 우분투 환경에서 USB 카메라를 사용해 보았는데, 장치 인식부터 RViz2에서 영상이 나오기까지 많은 어려움을 겪었다. 카메라를 정상적으로 실행하는 데만 한 시간이 넘게 걸리면서 ROS와 리눅스 환경에서 하드웨어를 다루는 것이 쉽지 않다는 것을 느꼈다.  
  시간이 부족해 신호등을 바운딩 박스로 분리하고 색상 인식 모델을 직접 만들지는 못한 점이 아쉬웠다. 나는 신호등 색상을 토픽으로 받아 해당 색상에 맞는 음성을 출력하는 역할을 맡았고, mp3 파일 준비부터 재생 방법, 토픽 구독까지 모두 직접 구현해 보았다. 이 과정을 통해 ROS 통신과 음성 출력 방식에 대해 이해할 수 있었다.  
  실제 신호등으로 실험해 본 결과, 초록불은 잘 인식되었지만 빨간불과 황색은 인식 속도가 느리거나 간혹 오인식되는 경우가 있었다. HSV 값 조정을 통해 개선할 수 있을 것이라 생각했으며, 실제 도로 환경에서 사용하려면 정확도를 더 높여야 한다는 점을 느꼈다.
